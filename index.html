Scalable Vector Graphics (SVG) is an XML markup language for describing two-dimensional vector graphics.
For Front-End Developers
We can create SVGs with code similar to XML and HTML.
For Designers
SVGs are vector-based. We won't end up with blurry or pixelated images, no matter our screen resolution.
SVG's are composed of different elements.
Those are probably the most recognizable elements you'll see if you inspect an SVG. If you check out the mozilla developer docs, you'll find a complete list. https://developer.mozilla.org/en-US/docs/Web/SVG/Element I'm not going to describe every element available but I do think it's worth while to attempt creating your own SVG's and inspecting the code generated by them by opening them up in Sublime or whatever text editor it is that you use.
Really, all that matters is finding a program that works for you and supports the .svg file format.
